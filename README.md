# 🚀 Urban Scooter – API Testing com Postman

## 📌 Sobre o projeto

Este repositório apresenta os testes de API realizados durante o Projeto **Urban Scooter**, desenvolvido no Bootcamp de QA da TripleTen.

O objetivo foi validar os serviços da aplicação através de testes nos endpoints da API REST, garantindo que as respostas, códigos de estado e regras de negócio estivessem de acordo com os requisitos definidos.

---

## 🎯 Objetivos

* Validar o funcionamento dos endpoints da API.
* Verificar códigos de estado HTTP.
* Validar respostas em formato JSON.
* Testar regras de negócio.
* Identificar e documentar falhas na API.

---

## 🛠 Ferramentas utilizadas

* Postman
* REST API
* JSON
* Git & GitHub

---

## 📋 Atividades realizadas

✔ Testes de endpoints REST

✔ Validação dos métodos HTTP (GET, POST, PUT e DELETE)

✔ Verificação de códigos de estado HTTP

✔ Validação da estrutura das respostas JSON

✔ Testes positivos e negativos

✔ Documentação de bugs encontrados

---

## 🔍 Cenários testados

### Testes Positivos

* Login com credenciais válidas.
* Criação de pedidos.
* Consulta de informações.
* Atualização de dados.
* Exclusão de registos.

### Testes Negativos

* Campos obrigatórios vazios.
* Dados inválidos.
* Formatos incorretos.
* Identificadores inexistentes.
* Métodos HTTP não permitidos.
